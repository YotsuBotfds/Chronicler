# Chronicler Audit - GPT-5 - 2026-03-31

Repo-wide audit focused on whether we should harden the codebase before starting `M60a`.

Scope:
- Python simulation and CLI
- Rust agent crate and Python/Rust boundary
- Viewer, bundle-loading surface, and repo organization
- Tests, validation workflow, and tooling hygiene

Method:
- Local verification plus six subagent passes across Python core, Rust/FFI, viewer/docs, validation/CLI, test infrastructure, and large-file design hotspots
- Spot-checked the highest-severity findings locally before including them here

## Executive Summary

The repo is healthy on its primary happy-path validation surface:
- `.venv\Scripts\python.exe -m pytest -q tests` -> `2242 passed, 4 skipped`
- `cargo test --quiet` in `chronicler-agents/` -> pass
- `cargo nextest run` in `chronicler-agents/` -> `762 passed, 2 skipped`
- `npm exec vitest run` in `viewer/` -> `83 passed`
- `npm run build` in `viewer/` -> pass, but warns about a `644.75 kB` client chunk

But that green surface is incomplete. The audit found several issues that are good reasons to pause before `M60a`:
- real correctness bugs in action resolution, war bookkeeping, and aggregate writeback
- hidden failing test surfaces that default commands do not run
- fail-open validation/CLI behavior that can misreport success
- stale viewer contract fixtures and viewer tooling drift
- growing architectural duplication in politics, FFI, and turn orchestration

Bottom line:
- I would not start `M60a` until a short repo-hardening tranche lands
- The mainline is not broken, but it is easier than it should be to get false confidence from green checks

## Priority Findings

### Blockers Before `M60a`

1. `TRADE` can mint treasury between civs that do not actually have an active route.
   Evidence: `src/chronicler/action_engine.py:208`, `src/chronicler/action_engine.py:633`, `src/chronicler/action_engine.py:751`, `src/chronicler/resources.py:134`.
   Why it matters: this is direct economic correctness drift. Eligibility uses relationship quality, not route existence, and `resolve_trade()` still pays out.

2. Executed action and recorded action can diverge.
   Evidence: `src/chronicler/action_engine.py:296`, `src/chronicler/simulation.py:559`, `src/chronicler/simulation.py:575`, `src/chronicler/simulation.py:1350`.
   Why it matters: `_resolve_war_action()` can fall back to `DEVELOP`, but `phase_action()` still records `war`. That poisons war weariness, peace momentum, streak logic, analytics, and any system reading `world.action_history`.

3. Normal wars do not initialize `war_start_turns`.
   Evidence: `src/chronicler/action_engine.py:305`, `src/chronicler/politics.py:732`, `src/chronicler/politics.py:876`.
   Why it matters: duration-sensitive congress and peace logic treat most wars as effectively new because only the federation-defense path stamps a start turn.

4. Civs that still control land but have zero live agents can disappear from Rust aggregates, leaving stale civilization stats on the Python side.
   Evidence: `chronicler-agents/src/pool.rs:659`, `chronicler-agents/src/pool.rs:682`, `src/chronicler/agent_bridge.py:2055`, `src/chronicler/agent_bridge.py:2087`.
   Why it matters: `_write_back()` rewrites stats only for civ rows that exist in the aggregate batch. Missing rows keep old military/economy/culture/stability values.

5. The promotions FFI does not carry authoritative polity identity.
   Evidence: `chronicler-agents/src/ffi.rs:2410`, `chronicler-agents/src/ffi.rs:2451`, `src/chronicler/agent_bridge.py:1418`, `src/chronicler/agent_bridge.py:1436`.
   Why it matters: Python reconstructs both `civilization` and `origin_civilization` from `origin_region`'s current controller, which is wrong after migration, conquest, secession, or loyalty changes.

6. The validation surface is fail-open.
   Evidence: `src/chronicler/validate.py:2014`, `src/chronicler/validate.py:2036`, `src/chronicler/validate.py:2055`, `scripts/m53b_run_validation.py:111`, `scripts/m53b_run_validation.py:122`.
   Why it matters: unknown oracles are silently ignored, `validate.py` always exits `0`, and the canonical runner treats subprocess success as validation success.

7. `--compare` is effectively inert unless paired with `--analyze`.
   Evidence: `src/chronicler/main.py:867`, `src/chronicler/main.py:1027`, `src/chronicler/main.py:1164`, `src/chronicler/main.py:1203`.
   Why it matters: `chronicler --compare baseline.json` can fall through to a normal simulation run instead of generating a delta report.

8. The default Python test command hides a stale, already-failing `test/` tree that still contains direct tests for live codepaths.
   Evidence: `pyproject.toml:25`, `test/test_m30_bridge.py:42`, `test/test_m36_culture.py:91`, `test/test_m38a_temples.py:53`.
   Why it matters: normal `pytest` never sees this tree, but those files still contain the most direct tests for `_process_promotions`, `_process_deaths`, `_detect_character_events`, conquest transitions, and temple prestige behavior.

### Strongly Recommended Before or Alongside `M60a`

9. Sequential batch API narration still accumulates tokens across seeds.
   Evidence: `src/chronicler/batch.py:89`, `src/chronicler/main.py:743`, `docs/superpowers/progress/phase-6-progress.md:706`.
   Why it matters: `run_batch()` reuses one narrative client across seeds, so per-seed bundle metadata can report cumulative token totals.

10. `--batch --parallel` silently drops real LLM clients.
    Evidence: `src/chronicler/batch.py:49`, `src/chronicler/batch.py:63`, `src/chronicler/batch.py:108`, `src/chronicler/narrative.py:970`, `README.md:103`.
    Why it matters: the parallel path always dispatches `_run_single_no_llm()`, which pushes `execute_run()` into dummy-client behavior for narrated runs.

11. The public resume path and the CLI resume path do not behave the same.
    Evidence: `src/chronicler/main.py:172`, `src/chronicler/main.py:261`, `src/chronicler/main.py:1207`.
    Why it matters: `execute_run()` recreates fresh `MemoryStream`s on resume, while `main()` explicitly reloads `memories_*.json`.

12. The pure-Python economy oracle crashes on valid barren regions.
    Evidence: `src/chronicler/economy.py:122`, `src/chronicler/economy.py:1172`.
    Why it matters: `bootstrap_region_stockpile()` already accepts `EMPTY_SLOT` (`255`), but `compute_economy()` still unconditionally resolves production from slot 0.

13. Black-market leakage only checks the first controlled region.
    Evidence: `src/chronicler/simulation.py:295`, `src/chronicler/simulation.py:311`.
    Why it matters: embargoed civs can miss valid smuggling routes if the first controlled region has none.

14. The severity-multiplier sweep is still incomplete.
    Evidence: `src/chronicler/simulation.py:168`, `src/chronicler/simulation.py:170`, `src/chronicler/simulation.py:323`, `src/chronicler/simulation.py:327`, `src/chronicler/simulation.py:875`.
    Why it matters: some negative effects still bypass `get_severity_multiplier(...)`, including environment plague stability loss and ongoing-war bankruptcy stability drain.

15. Viewer lint is red even though viewer tests and build are green.
    Evidence: `viewer/src/components/TerritoryMap.tsx:204`, `viewer/src/components/TerritoryMap.tsx:210`, `viewer/src/hooks/useLiveConnection.ts:44`, `viewer/src/hooks/useTimeline.ts:32`.
    Why it matters: the viewer can ship with passing tests/build but still violate the intended React usage rules. This is exactly the kind of "looks green, isn't healthy" signal we should clean up.

16. The committed viewer fixture is stale relative to the live Python bundle contract.
    Evidence: `viewer/src/__fixtures__/sample_bundle.json`, `scripts/generate_viewer_fixture.py:1`, `viewer/src/lib/bundleLoader.ts:4`, `viewer/src/components/__tests__/components.test.tsx:18`.
    Why it matters: current viewer tests mostly shallow-cast the fixture and do not enforce Python->viewer contract fidelity.

17. Analytics CLI/reporting has drifted behind the extractor inventory.
    Evidence: `src/chronicler/analytics.py:1183`, `src/chronicler/analytics.py:1257`, `src/chronicler/analytics.py:1298`, `src/chronicler/analytics.py:1546`, `src/chronicler/analytics.py:1744`, `src/chronicler/analytics.py:1879`, `src/chronicler/analytics.py:1952`.
    Why it matters: later extractors exist but are not surfaced through `--analyze`, and the human-readable politics formatter still looks for stale keys like `federation_rate` instead of `federation_formed_rate`.

   Evidence: `chronicler-agents/src/behavior.rs:1003`.

## Recommended Pre-`M60a` Sequence

1. Close the correctness bugs first.
   Scope:
   - route-gate `TRADE`
   - record resolved action rather than requested action
   - initialize and clear `war_start_turns` on the main war lifecycle
   - fix aggregate writeback for zero-agent controller civs
   - carry authoritative civ identity in promotions FFI
   - finish the missing severity-multiplier consumers

2. Make validation and CLI behavior fail-closed.
   Scope:
   - unknown oracle names should error
   - validation should return non-zero on failure or invalid input
   - `--compare` should be a real top-level mode or be removed
   - `scripts/m53b_run_validation.py --profile full` should force the documented defaults
   - document that parallel narrated batches are not supported, or fix them

3. Resolve the hidden red surfaces.
   Scope:
   - decide whether `test/` is legacy archive or active test suite
   - migrate any still-relevant cases into `tests/`
   - delete or archive the rest
   - fix viewer lint
   - add `npm test`
   - decide whether `cargo clippy -D warnings` is a target now or later, then codify that decision

4. Re-establish the viewer contract.
   Scope:
   - regenerate `viewer/src/__fixtures__/sample_bundle.json`
   - add a Python->viewer contract check that regenerates and parses the fixture
   - route all viewer-side bundle/JSON parsing through one loader surface

5. Clean the analytics and reporting surface.
   Scope:
   - wire the newer analytics extractors into `--analyze`
   - fix stale politics key names in the text report
   - decide which metrics are canonical before more milestones depend on them

6. Schedule a contained design-debt wave before the next large milestone jump.
   Scope:
   - choose one politics source of truth
   - split `AgentBridge`
   - split `ffi.rs`
   - introduce typed per-phase contexts instead of `world._*` scratch channels
   - split `validate.py` into IO/models/oracles
